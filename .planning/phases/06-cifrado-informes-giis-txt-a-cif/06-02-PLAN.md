# Plan 06-02 — Frontend: Etiquetas de descarga CEX Cifrado, CEX.txt, LES Cifrado, LES.txt

**Objetivo:** Actualizar la vista de Exportación GIIS para mostrar 4 enlaces de descarga: "CEX Cifrado" (ZIP), "CEX.txt" (TXT), "LES Cifrado" (ZIP), "LES.txt" (TXT). Orden: ZIP (Cifrado) principal, TXT secundaria. CDT eliminado (solo CEX y LES). Sin avisos adicionales.

**Contexto (06-CONTEXT.md):** Etiquetas exactas. Prioridad: ZIP primero, TXT segundo. Solo guías CEX y LES.

---

## must_haves

- [ ] **Etiquetas de enlaces:** Por cada guía (CEX, LES), mostrar 2 enlaces: "CEX Cifrado" y "CEX.txt" / "LES Cifrado" y "LES.txt". Reemplazar las etiquetas actuales ("Descargar CEX", "Descargar LES" o similar).
- [ ] **Orden:** Para cada guía, el enlace "Cifrado" (ZIP) aparece primero, "txt" (TXT) segundo.
- [ ] **URLs de descarga:** "CEX Cifrado" / "LES Cifrado" → GET `api/giis-export/batches/:batchId/download-deliverable/:guide`. "CEX.txt" / "LES.txt" → GET `api/giis-export/batches/:batchId/download/:guide`.
- [ ] **Solo CEX y LES:** No mostrar enlaces para CDT. Si el listado de artifacts incluye otros guides, filtrar a CEX y LES únicamente.
- [ ] **Condición de visibilidad:** Los enlaces "Cifrado" se muestran solo si `artifact.zipPath` existe (batch completado con cifrado). Los enlaces "txt" se muestran si `artifact.path` existe (TXT generado). Para batches Phase 6+, ambos estarán disponibles al completar.
- [ ] **Comportamiento descarga:** Descarga directa en navegador (target="_blank" o uso de blob/link download según implementación actual). Sin abrir nueva pestaña innecesaria si la app ya maneja descarga inline.
